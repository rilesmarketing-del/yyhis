# 管理端账号与角色查看页设计

## 背景

当前系统已经具备最小鉴权能力，后端有真实演示账号、Token 登录和角色路由控制；但 [AdminAuth.vue](/C:/Users/89466/Desktop/yy/frontend/src/views/admin/AdminAuth.vue) 仍然是静态角色与权限假数据页，无法反映系统内真实账号状态，也无法让管理端可视化查看当前演示账号与角色边界。

## 目标

将 `AdminAuth` 改造成一个从简、真实、只读的“账号与角色查看页”：

- 展示真实演示账号列表
- 展示按角色聚合的账号概览
- 仅允许 `admin` 访问
- 不提供账号新增、编辑、删除
- 不提供权限矩阵编辑

## 方案选择

### 方案 A：只读账号与角色查看页

- 新增 `GET /api/admin/auth/accounts`
- 返回真实账号列表与角色概览
- 前端展示账号表与角色说明面板

优点：

- 口径真实
- 改动最小
- 与当前“最小鉴权”阶段匹配

缺点：

- 还不是完整权限管理中心

### 方案 B：账号列表加可编辑权限矩阵

优点：

- 更像正式后台

缺点：

- 当前没有完整 RBAC 模型，容易做成半真半假

### 方案 C：继续保留静态权限页

优点：

- 无风险

缺点：

- 与当前真实鉴权底座脱节

最终采用方案 A。

## 范围

### 本轮包含

- 新增 admin 只读账号接口
- 展示真实账号列表
- 展示角色汇总和角色能力说明
- AdminAuth 页面真实化

### 本轮不包含

- 账号新增、编辑、停用
- 角色创建、删除
- 权限点编辑
- Token 会话管理页面

## 接口设计

### `GET /api/admin/auth/accounts`

仅允许 `admin` 访问。

返回两块数据：

- `accounts`
  - `username`
  - `displayName`
  - `role`
  - `patientId`
  - `enabled`
- `roleSummary`
  - `role`
  - `label`
  - `count`
  - `scopeHint`

## 数据口径

- 账号数据来自 `UserAccount`
- 角色汇总按 `UserRole` 聚合生成
- 角色说明先固定为最小真实口径：
  - `patient`：患者端访问
  - `doctor`：医生端接诊与病历
  - `admin`：管理端排班、总览与报表

## 前端展示

- 左侧：真实账号表
- 列：账号、显示名、角色、患者绑定、状态
- 右侧：角色概览与当前选中账号角色说明
- 点击账号行后，右侧切换到对应角色信息
- 页面保留刷新能力
