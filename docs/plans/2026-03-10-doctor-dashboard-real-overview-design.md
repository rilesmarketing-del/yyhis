# 医生首页真实总览设计

## 背景

当前医生端已经完成了接诊台、病历页、患者列表和当前接诊工作台的真实联动，但 [DoctorDashboard.vue](/C:/Users/89466/Desktop/亿元项目/frontend/src/views/doctor/DoctorDashboard.vue) 仍然是静态演示页，统计卡、候诊概览和待处理事项都没有接入真实数据。

## 目标

将医生首页升级为最小真实总览页：

- 统计卡基于真实待接诊队列、接诊记录、患者列表生成
- 主表展示今日候诊与接诊概览
- 侧栏展示真实待处理事项
- 不新增后端接口，只复用现有医生端数据接口

## 方案选择

### 方案 A：前端聚合真实医生数据

- 直接复用 `fetchDoctorQueue`、`fetchDoctorRecords`、`fetchDoctorPatients`
- 在前端聚合统计卡、概览表、提醒
- 不新增后端聚合接口

优点：

- 推进最快
- 风险最低
- 适合当前迭代节奏

缺点：

- 首页统计口径暂时只覆盖已接入的医生业务

### 方案 B：后端新增聚合接口

优点：

- 后续扩展更整齐

缺点：

- 本轮明显过重
- 会扩大后后端改动面

### 方案 C：只改统计卡，其他区域继续静态

优点：

- 实现更快

缺点：

- 页面观感割裂
- 不值得单独做

最终采用方案 A。

## 范围

### 本轮包含

- 新增医生首页前端聚合逻辑
- 真实统计卡
- 真实“今日候诊与接诊概览”表格
- 真实“待处理事项”列表
- 快捷入口跳转到接诊台、病历页、工作台、患者列表

### 本轮不包含

- 新增后端接口
- 图表组件
- 更复杂的日/月趋势分析
- 改动医生端现有业务状态机

## 数据来源

页面数据复用现有接口：

- `GET /api/doctor/clinic/queue`
- `GET /api/doctor/records`
- `GET /api/doctor/patients`

## 展示规则

### 统计卡

- `今日接诊`：今天日期的 `IN_PROGRESS + COMPLETED` 接诊记录数
- `当前候诊`：当前待接诊队列人数
- `待完成病历`：`IN_PROGRESS` 记录数
- `累计接诊患者`：医生患者列表人数

### 今日候诊与接诊概览

- 数据由待接诊队列和今日接诊记录合并得到
- 字段：患者、科室、时间、状态、来源
- 状态：
  - 队列记录显示 `待接诊`
  - `IN_PROGRESS` 显示 `接诊中`
  - `COMPLETED` 显示 `已完成`

### 待处理事项

优先生成 3 到 5 条真实事项：

- 当前有 X 位患者待接诊
- 有 X 份病历待完成
- 今日已完成 X 次接诊
- 如果没有任何待处理项，则显示“当前工作已清空，可查看历史患者”

### 快捷入口

- 去接诊台 `/doctor/clinic`
- 去病历页 `/doctor/records`
- 去工作台 `/doctor/orders`
- 去患者列表 `/doctor/patients`

## 测试策略

前端先补纯映射脚本，覆盖：

- 统计卡计算正确
- 概览表能正确合并队列和今日接诊记录
- 待处理事项能按真实数据生成

然后执行：

- `node ./scripts/test-doctor-dashboard.mjs`
- `npm run build`
- `npm run verify:bundle`
